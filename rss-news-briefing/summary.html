1 POC 한 줄 정의

한국 주요 언론사 RSS 피드를 수집하여 42개 키워드로 자동 필터링·요약한 뒤, 노션 DB에 저장하고 뉴스레터·급상승 알림·주간 리포트·심층 브리핑을 자동 생성하는 시스템
INPUT 입력
RSS 피드 10개 (연합뉴스, 매일경제, 한국경제, 지디넷, 조선, 경향)
↓
PROCESS 처리
키워드 매칭(42개) → 중복 제거 → 카테고리/중요도 분류
↓
OUTPUT 출력
노션 DB 저장 + 뉴스레터 + 급상승 알림 + 주간 리포트 + 심층 브리핑

2 사용 기술

도구용도
Node.js + rss-parserRSS 피드 수집 및 XML 파싱
Claude Code전체 파이프라인 구축, 코드 생성

6 확장 기능 (4개)

A 일일 뉴스레터 초안
매일 수집된 기사를 "헤드라인 + 카테고리별 1줄 요약 + 급상승 키워드" 형식으로 자동 작성. 노션 페이지에 저장하여 바로 공유 가능.
실행 주기: 매일 08:00
D 키워드 급상승 알림
오늘 키워드 등장 횟수를 최근 7일 평균과 비교. 2배 이상 급증한 키워드를 자동 감지하여 태깅.
실행 주기: 매일 수집 직후 자동
C 주간 트렌드 리포트
일주일간 축적된 기사를 분석하여 "Top 5 키워드 + 카테고리별 핵심 이슈 3개"를 한 페이지로 정리.
실행 주기: 매주 일요일
G 요일별 심층 브리핑
요일마다 다른 카테고리를 집중 분석.
월=경제, 화=정치, 수=기술, 목=국제, 금=스포츠
실행 주기: 평일 매일 (요일별 카테고리 자동 전환)

7 프로젝트 구조

rss-news-briefing/
├── config.js ← RSS 10개, 키워드 42개, 노션 DB ID
├── rss-fetcher.js ← RSS 수집 + 키워드 매칭 + 중복 제거
├── notion-helpers.js ← 뉴스레터/급상승/리포트/브리핑 생성
├── run.js ← 메인 실행 (daily/weekly)
├── notion-sync.js ← 노션 MCP 연동
├── package.json ← npm scripts
└── output/ ← 결과물 저장 폴더
    ├── *-articles.json ← 필터링된 기사
    ├── *-newsletter.md ← 뉴스레터 초안
    ├── *-surges.json ← 급상승 키워드
    ├── *-weekly-report.md ← 주간 리포트
    ├── *-deep-*.md ← 심층 브리핑
    └── history.json ← 7일 키워드 히스토리
